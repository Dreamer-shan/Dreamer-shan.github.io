定义

　并查集是一种树型的数据结构，用于处理一些 不相交集合的合并及查询问题（即所谓的并、查）。比如说，我们可以用并查集来判断一个森林中有几棵树、某个节点是否属于某棵树等。并查集的主要构成：

  • 一个整型数组 parent[]和函数 init()、find()、join()、isConnected()构成
  • 数组 parent[] 记录了每个点的前驱节点是谁，函数 find(x) 用于查找指定节点 x 属于哪个集合，函数 join(x,y) 用于合并两个节点 x 和 y ，isConnected(x,y)用于判断 x 和 y 是否在同一个子树（集合）中。
init(int size)初始化函数
find(int x)函数

　find(int x) 函数是寻找 x 的根节点，如果节点 x 是根节点，那么它的前驱应该是它自己，也就是parent[x] == x，如果不满足该条件，说明此时没找到根节点，继续向上查找。

　join() 函数是合并两个节点。

路径压缩

　join(int x，int y) 可以合并两个节点。但是问题也随之产生，我们并没有规定合并后谁做根节点。

　这里并没有规定谁是谁的前驱的规则，其实 parent[xRoot]=yRoot; 也是可以的。但这样就导致了最终的树状结构无法预计，即有可能是良好的 n 叉树，也有可能是单支树结构（类似链表）。如果形成了单支树的结构，那么树的高度就会较大，查找性能必定会下降。

　最理想的情况就是所有结点的直接前驱都是根节点，这样一来整个树的结构就只有两级，此时 find 方法中的 while()中的代码最多只需要执行一次。因此，这就产生了路径压缩算法。

基于节点个数 size 的路径压缩

　使用 size 优化，也就是在 join 操作时， 节点数较小的树指向节点数较大的树。假设一棵大树与一棵小树进行合并时，如果是大树指向小树，那么需要找到大树的根节点，而大树的根节点通常深度较高，操作起来更费时。为此，需要额外的一个 size 数组，用来存储树的节点个数。

基于节点深度 rank 的路径压缩

　使用 rank 优化，也就是在 join 操作时，深度较小的树指向节点数较大的树。大树的根节点深度较高，操作起来更费时。为此，需要额外的一个 rank 数组，用来存储树的深度。合并两棵树之前需要获取两棵树的深度，深度较小的树指向深圳较大的树，并且需要更新新的树的深度。具体的说是如果两棵树深度不一样，合并后的深度是原本深度较大的树的深度。若果两棵树深度相等，则合并后的深度是原本深度 +1
